# Спецификация продукта — To-Do приложение

## Цель
Дать одному пользователю простой инструмент для ведения списка дел: быстро
добавить задачу, отметить её выполненной и удалить ненужное. Продукт намеренно
минимален — без аккаунтов, без базы данных, без внешних сервисов.

## Пользователи
- **Один локальный пользователь.** Запускает приложение у себя и работает с
  личным списком задач через браузер.
- Сценарий: открыл одну веб-страницу, увидел свои задачи, добавил/отметил/удалил.
- Многопользовательность и совместный доступ не предусмотрены.

## Фичи
1. **Добавление задачи** — форма с текстовым полем; новая задача попадает в
   список со статусом «не выполнено».
2. **Отметка выполнения** — чекбокс у задачи переключает флаг `done`
   (выполнено / не выполнено).
3. **Удаление задачи** — кнопка у задачи удаляет её из списка.
4. **Просмотр списка** — все задачи отображаются на одной веб-странице.
5. **Хранение на диске** — список задач сохраняется в JSON-файл и
   восстанавливается при следующем запуске.

### Модель данных
Файл хранилища — JSON-массив объектов задачи. Поля каждой задачи:
- `id` — уникальный идентификатор задачи;
- `text` — текст задачи (строка);
- `done` — флаг выполнения (`true` / `false`).

## Ограничения
- **Стек:** Python 3 + Flask. Зависимости только из `requirements.txt`
  (`flask`, `pytest`) — новые не вводить.
- **Хранилище:** один JSON-файл на диске. Без базы данных.
- **Без авторизации:** ни логина, ни паролей, ни сессий.
- **Интерфейс:** одна веб-страница. Только форма добавления и список задач с
  чекбоксом «выполнено» и кнопкой удаления — без счётчиков и фильтров.
- **Валидация ввода:** текст задачи обязателен и не должен быть пустым;
  максимальная длина — 200 символов. Пустой или превышающий лимит текст
  отклоняется с сообщением об ошибке.
- **Поведение хранилища при старте:** если JSON-файл отсутствует — создаётся
  пустой список. Если файл повреждён (невалидный JSON) — приложение аварийно
  завершается с явной ошибкой, чтобы не затереть данные.
- **Архитектурные правила** (`docs/rules/architecture.md`): docstring на каждую
  функцию, тест на каждую публичную функцию, `logging` вместо `print()`,
  файлы не длиннее 200 строк.

## Out of scope
Сознательно не входит в этот продукт:
- Авторизация, регистрация, мультипользовательский режим.
- База данных (SQLite, Postgres и т.п.).
- Редактирование текста уже созданной задачи.
- Дополнительные поля задачи: дата создания, приоритет, дедлайн, теги.
- Элементы UI: счётчик активных задач, фильтр по статусу, кнопка «очистить
  выполненные», сортировка, поиск.
- Деплой в облако, контейнеризация, CI/CD.
- REST API для внешних клиентов.
