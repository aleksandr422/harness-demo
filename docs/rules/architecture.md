# Архитектурные правила

## Запрещено
- Пароли/ключи в коде (использовать .env)
- print() для логирования (использовать logging)
- Файлы длиннее 200 строк
- Импорты из tests/ в src/

## Обязательно
- Docstring на каждую функцию
- Тест на каждую публичную функцию
- snake_case для переменных, CamelCase для классов
