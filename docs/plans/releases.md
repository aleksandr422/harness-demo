# Releases

История релизов To-Do приложения.

## v0.1.0 — 2026-05-21

Первый рабочий релиз: минимальное To-Do приложение на Python + Flask с
хранением задач в JSON-файле.

### Что вошло

Реализованы все задачи `docs/plans/execplan.md`:

- **TASK-001** — `src/storage.py`: слой хранения в JSON-файле (атомарная
  запись, пустой список при отсутствии файла, аварийное завершение при
  повреждённом JSON).
- **TASK-002** — `src/service.py`: бизнес-логика — валидация текста,
  add / delete / toggle, генерация уникального `id`.
- **TASK-003** — `src/routes.py`: HTTP-маршруты (Flask Blueprint) —
  `GET /`, `POST /add`, `POST /toggle/<id>`, `POST /delete/<id>`.
- **TASK-004** — `src/templates/index.html`: одностраничный веб-интерфейс.
- **TASK-005** — `src/app.py`: точка входа, фабрика `create_app()`.

Тесты: `tests/` — 24 pytest-теста, все зелёные. Линтер `scripts/lint.sh` —
зелёный.

### Проверка деплоя (Пайплайн 05)

- `bash scripts/lint.sh` — зелёный.
- `pytest` — 24 passed.
- Локальный запуск + HTTP-проверки: GET `/` (200, форма видна),
  добавление / отметка / удаление задачи, отклонение пустого текста и
  текста длиннее 200 символов — все проверки пройдены.

### Известные ограничения

- Dev-сервер Flask — не для production (нужен WSGI-сервер).
- Нет защиты от конкурентного доступа: каждая мутация выполняет
  load → mutate → save без блокировки файла. Рассчитано на одного
  локального пользователя.
- `SECRET_KEY` обязателен: без переменной окружения `TODO_SECRET_KEY`
  flash-сообщения упадут с `RuntimeError`.
- `load_tasks` проверяет только то, что JSON — список; структура полей
  задач не валидируется.
- `id` может переиспользоваться после удаления последней задачи.
- Операция с несуществующим `id` (`toggle`/`delete`) проходит молча, без
  сообщения пользователю.

Подробности — `docs/plans/review_notes.md`.

### Команды запуска

```bash
pip install -r requirements.txt
TODO_SECRET_KEY=dev-secret python src/app.py
```

Приложение поднимается на http://127.0.0.1:5000

Опциональная переменная `TODO_TASKS_FILE` задаёт путь к файлу хранилища
(по умолчанию — `tasks.json` в корне проекта).
