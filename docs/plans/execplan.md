# ExecPlan — To-Do приложение

Источник: `docs/specs/product.md`. План разбивает спецификацию на атомарные
задачи (≤ 1 модуль на задачу). Порядок — снизу вверх по зависимостям:
хранилище → сервис → routes → шаблон → точка входа.

Тесты не входят в задачи этого плана — их пишет Пайплайн 04 (Review) на каждую
публичную функцию модулей `src/`.

---

## TASK-001 — Слой хранения (JSON-файл)

- **Описание:** модуль работы с JSON-файлом задач. Загружает список задач при
  старте и сохраняет его на диск. Если файла нет — возвращает пустой список;
  если файл повреждён (невалидный JSON) — аварийно завершает работу с явной
  ошибкой, не затирая данные.
- **Файлы:** `src/storage.py` (создаёт).
- **Зависимости:** нет.
- **Definition of Done:**
  - Константа пути к файлу хранилища.
  - `load_tasks()` — возвращает список задач; нет файла → `[]`; битый JSON →
    исключение/аварийное завершение с понятным сообщением.
  - `save_tasks(tasks)` — атомарно (через временный файл) пишет список в JSON.
  - Логирование через `logging`, без `print()`.
  - Docstring на каждой функции.
  - `bash scripts/lint.sh` проходит зелёным.

- **Статус:** [x]

---

## TASK-002 — Сервисный слой (бизнес-логика)

- **Описание:** логика управления задачами поверх слоя хранения. Генерация
  уникального `id`, валидация текста, добавление / удаление / переключение
  статуса `done`. Каждая операция, меняющая данные, сохраняет их через
  `storage.save_tasks`.
- **Файлы:** `src/service.py` (создаёт).
- **Зависимости:** TASK-001.
- **Definition of Done:**
  - `get_tasks()` — возвращает текущий список задач.
  - `add_task(text)` — валидирует текст (непустой, ≤ 200 символов, иначе
    `ValueError`), создаёт задачу `{id, text, done=False}` с уникальным `id`.
  - `delete_task(task_id)` — удаляет задачу по `id`.
  - `toggle_task(task_id)` — переключает флаг `done`.
  - Текст обрезается по краям (`strip`); валидация вынесена в отдельную функцию.
  - Docstring на каждой функции, логирование через `logging`.
  - `bash scripts/lint.sh` проходит зелёным.

- **Статус:** [x]

## TASK-003 — HTTP-маршруты (Flask Blueprint)

- **Описание:** Flask-маршруты, связывающие веб-запросы с сервисным слоем.
  Оформить как `Blueprint`. Ошибки валидации показывать пользователю
  (flash-сообщение), после POST — redirect на главную (PRG-паттерн).
- **Файлы:** `src/routes.py` (создаёт).
- **Зависимости:** TASK-002.
- **Definition of Done:**
  - `GET /` — рендерит шаблон со списком задач.
  - `POST /add` — добавляет задачу; при `ValueError` показывает ошибку.
  - `POST /toggle/<task_id>` — переключает статус задачи.
  - `POST /delete/<task_id>` — удаляет задачу.
  - Все POST-маршруты завершаются redirect на `GET /`.
  - Docstring на каждой функции, логирование через `logging`.
  - `bash scripts/lint.sh` проходит зелёным.

- **Статус:** [x]

## TASK-004 — HTML-шаблон главной страницы

- **Описание:** единственная веб-страница: форма добавления задачи и список
  задач. У каждой задачи — чекбокс «выполнено» (POST на `/toggle`) и кнопка
  удаления (POST на `/delete`). Блок вывода ошибок валидации (flash).
- **Файлы:** `src/templates/index.html` (создаёт).
- **Зависимости:** TASK-003 (контракт переменных шаблона задаёт `routes.py`).
- **Definition of Done:**
  - Форма добавления отправляет POST на `/add`.
  - Список задач: текст, чекбокс статуса, кнопка удаления у каждой.
  - Выполненные задачи визуально отличаются (например, зачёркнутый текст).
  - Отображаются flash-сообщения об ошибках.
  - Корректно отображается пустой список.
  - Файл лежит в `src/templates/` (Flask находит его по умолчанию).

- **Статус:** [x]

## TASK-005 — Точка входа (Flask-приложение)

- **Описание:** сборка приложения: фабрика `create_app()`, регистрация
  blueprint из `routes.py`, конфигурация, запуск dev-сервера.
- **Файлы:** `src/app.py` (создаёт).
- **Зависимости:** TASK-003, TASK-004.
- **Definition of Done:**
  - `create_app()` создаёт экземпляр Flask и регистрирует blueprint.
  - `SECRET_KEY` для flash берётся из переменной окружения (не хардкод в коде).
  - Блок `if __name__ == "__main__":` запускает dev-сервер.
  - Приложение поднимается, главная страница открывается в браузере, все
    операции (добавить / отметить / удалить) работают end-to-end.
  - Docstring на каждой функции, логирование через `logging`.
  - `bash scripts/lint.sh` проходит зелёным.

- **Статус:** [x]
