# Пайплайн 02 — Plan: Спецификация → ExecPlan

## Цель
Превратить спецификацию в исполнимый план разработки (КАК строим).

## Вход
- Файл `docs/specs/product.md`.

## Шаги
1. Прочитать AGENTS.md и `docs/specs/product.md`.
2. Разбить спецификацию на атомарные задачи (макс. 1 файл на задачу).
3. Записать задачи в `docs/plans/execplan.md` как чек-лист.
4. Каждая задача содержит:
   - **ID** — уникальный идентификатор (TASK-001, TASK-002, ...).
   - **Описание** — что нужно сделать.
   - **Файлы** — какие файлы создаются/меняются.
   - **Зависимости** — от каких других задач зависит.
   - **Критерии готовности** (Definition of Done) — когда задача считается
     выполненной.

## Выход
- Файл `docs/plans/execplan.md` с чек-листом задач.

## Как запустить
Попросить агента выполнить пайплайн 02 после готовности спецификации.
