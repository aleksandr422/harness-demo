## Известный технический долг

Открытые наблюдения из `docs/plans/review_notes.md` (некритичные):

- **#1 SECRET_KEY=None.** `create_app()` не предупреждает, если переменная
  `TODO_SECRET_KEY` не задана; первый `flash` упадёт с `RuntimeError`.
  Стоит логировать warning при старте.
- **#4 load_tasks не валидирует структуру задач.** Проверяется только то,
  что JSON — список; поля `id/text/done` не валидируются.
- **#2 Переиспользование id** после удаления последней задачи (`max+1`).
- **#5 Нет обратной связи** при `toggle`/`delete` несуществующего id —
  маршрут молча редиректит.
- **#3 Нет защиты от конкурентного доступа** (load → mutate → save без
  блокировки) — приемлемо для одного пользователя по спецификации.

## Потенциальные улучшения на будущее

- **Линтеры.** Заменить самописный `scripts/lint.sh` на `ruff` + `black`
  (форматирование, неиспользуемые импорты, стиль) — поймало бы мёртвый
  `logger` автоматически.
- **CI.** GitHub Actions: прогон `lint.sh` + `pytest` на каждый push/PR.
- **Контейнеризация.** `Dockerfile` и `docker-compose.yml` для запуска
  приложения одной командой, без ручной установки зависимостей.
- **E2E UI-тесты.** Playwright-сценарии на реальный браузер (добавление,
  отметка, удаление, отображение ошибок валидации) поверх unit-тестов.
- **Покрытие.** Подключить `pytest-cov` и порог покрытия в линтере.
- **Структура пакета.** Оформить `src/` как Python-пакет (`__init__.py`)
  вместо плоских модулей с абсолютными импортами через `sys.path`.
