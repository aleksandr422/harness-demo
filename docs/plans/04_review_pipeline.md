# Пайплайн 04 — Review: Код → Review

## Цель
Проверить качество реализованного кода и покрыть его тестами.

## Вход
- Код в `src/`.
- Файл `docs/rules/architecture.md`.

## Шаги
1. Прочитать AGENTS.md и `docs/rules/architecture.md`.
2. Написать pytest-тесты в `tests/` на каждую публичную функцию.
3. Запустить `pytest`.
4. Проверить код на соответствие правилам из `architecture.md`.
5. Записать замечания в `docs/plans/review_notes.md`.
6. Исправить найденные проблемы и повторить проверку.

## Выход
- Тесты в `tests/`.
- Файл `docs/plans/review_notes.md` с замечаниями.

## Как запустить
Попросить агента выполнить пайплайн 04 после завершения пайплайна 03.
